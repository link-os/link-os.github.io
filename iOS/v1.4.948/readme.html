Link-OS Multiplatform SDK and Zebra Utilities

Build: 1.4.948

1. Installation

The SDK and Zebra Utilities are supported on the following devices:

  • iPod touch™ 4, 5
  • iPad™ 2, 3, 4, Air, Air 2
  • iPad™ Mini 1, 2, 3
  • iPhone® 5
  • iPhone® 5C
  • iPhone® 5S
  • iPhone® 6
  • iPhone® 6 Plus

iOS versions:

  • 5.0 and later

The SDK and Zebra Utilities are supported on the following printers:

  • Mobile - iMZ™, MZ™, QL Plus™, QLn™ series, RW™ series, P4T™/RP4T™
  • Desktop - G-Series™, HC100™, TLP/LP 2824 Plus™
  • High-Performance/Midrange - Xi4™ and XiIIIPlus™, PAX4™ series, ZM400™/ZM600™, S4M™, 105SL™
  • RFID - RXi™, R110PAX4™, RZ400™/RZ600™
  • Kiosk - KR403™

SDK Developed using Xcode 5.0.1.

2. Requirements

  1. Xcode 4.5 and higher
  2. Minimum iOS SDK is 5.0
  3. Valid architectures are armv7, armv7s, and arm64

3. Tips and Advice

  1. As a best pracitce, Zebra recommends not making calls to our API from the GUI thread. Apple recommends using Grand Central Dispatch to accomplish this.
  2. Each ZebraPrinterConnection object should only be used on a single thread.

4. Known Issues

  1. Printing from the 'Traffic Ticket' or 'Asset Tag' demos from the Zebra Utilities sets the printer page width to 2 inches. All subsequent labels will be printed with a 2 inch width. Reset the printer width to resolve this issue.
  2. Printing large documents from Zebra Utilities may cause the application to quit due to memory constraints
